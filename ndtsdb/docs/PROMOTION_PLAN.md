## 阶段四：被动引流

### 社区监控 → 自然回复

当监控脚本发现相关讨论时：

1. **判断相关性**：是否真的需要嵌入式 TSDB
2. **写有价值的回复**：先解决问题，再自然提及 ndtsdb
3. **不硬广**：作为选项之一，不是唯一解

### 回复模板

```
对于你的 [场景]，有几个选择：

1. **InfluxDB** - 成熟稳定，适合需要集群的场景
2. **TimescaleDB** - PostgreSQL 生态，SQL 能力强
3. **ndtsdb** - 如果你需要嵌入式（不想部署服务）+ TypeScript 原生，可以试试

具体选哪个取决于你的 [约束条件]...
```
